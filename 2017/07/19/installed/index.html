主页搭建全记录

Jul 19, 2017 | 技术 | 阅读

趁着域名还在备案，记录下主页搭建过程。
部署在github.io上的过程主要参考luckykun的博文

本地开发环境配置

  • 安装git(ver 2.13.3)
  • 安装node.js(ver 8.1.3)与npm(ver 5.0.3，node.js自带)
  • 通过npm安装hexo(ver 1.0.3)
  • git自己喜欢的hexo主题到本地文件夹如hexo，如Cho的maupassant

    git clone git clone https://github.com/tufu9441/maupassant-hexo.git themes/maupassant
    npm install hexo-renderer-jade@0.3.0 –save
    npm install hexo-renderer-sass –save`

至此，本地开发环境配置完毕，修改目录下的hexo/_config.yml和主题下的hexo/themes/maupassant/_config.yml以及layout下的布局以满足自身需求。

发布博文只需在hexo/source/_posts/目录下添加md文件即可。

本地测试与部署至github pages

  • hexo generate //生成静态页面
  • hexo server //启动本地服务调试
  • hexo deploy //部署到指定站点

部署前需修改hexo/下_config.yml文件

deploy:
type: git
repository: https://github.com/gaonii/gaonii.github.io.git
branch: master

部署至服务端

首先是安装服务器nginx(ver 1.9.0)
安装并配置nginx.conf，我安装的目录在/nginx/conf

#如果站点没有访问权限，请添加：
user root;

…

server {
listen 80;
root /root/gaonii_blog/public;
index index.html index.htm;
server_name gaonii.cc www.gaonii.cc;
…

hexo静态页面生成在public文件夹下，只需上传内容至服务端即可。

在/nginx/sbin下测试nginx.conf是否正确

./nginx -t

重启nginx

./nginx -s reload

ok!
