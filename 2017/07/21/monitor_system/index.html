通过Django搭建设备监测系统

Jul 21, 2017 | 技术 | 阅读

这本来是上学期软件工程课的课堂大作业，名曰设备云平台监测系统设计。当时由于时间紧张就没有动手去开发，导致平时成绩被评为全班最后一名，差点挂科(好险好险)…最近入坑Django就写了一个小Demo练练手。该系统实现了将客户端所采集的信息，传输到服务器上，并实时显示。系统流程图什么的我就懒得做了，期末快看吐了都。

客户端：

Windows下的信息采集通过Python3.4调用wmi库获取PC信息。作为示例程序，我仅采集了CPU的实时利用率，其他一些参数如温度、硬盘占用空间等，可参考MSDN的wmi使用手册。当然了，还要设计好对应的数据库关系，这很重要，不过这次数据库设计并不是我们关心的重点。linux下的信息采集应该更容易，而且还能获取一些内核参数，这里我没有实现，请参考网络资源。

服务端：

使用Django1.11.2开发，通过nginx1.9.0+uWSGI2.0.15部署至阿里ecs，系统环境为ubuntu14.04。前端界面使用bootstrap框架。图表绘制采用highcharts，为了实现图表动态地实时更新，通过ajax定时读取数据库。请参考Highcharts官方文档教程，写的相当详细，比eCharts不知道高到哪里去。

由于域名还在备案，所以目前还只能用ip访问主页http://106.14.15.58/cmdb/monitor/
还有就是我并没有把该系统做成daemon，所以需要手动输入指令启动。

uwsgi –ini /root/Monitor/simplecmdb/uwsgi.ini

如果想部署成后台服务的话，用supervisor写个后台脚本就可以了。

左侧导航栏为当前数据库中ip按钮列表，点击ip按钮可获得主机详细数据(其实并不详细)，由于我的客户端测试环境在一个局域网下，所以采用局域网ip识别。想要用公网ip就在客户端Python源代码中修改，很容易。
